# ID: 1
- PBI名: チェックリスト登録機能追加
- ステータス: done
- ユーザストーリー
  - ユーザは手動でチェックリストを登録したり、編集することができる
  - レビューを実行するためにはレビュースペースにチェックリストを登録する必要があるからだ
- 受け入れ基準
  - レビュー対象一覧画面（`05b-review-target-list`参考）が作成されている
    - レビュー対象については、レビュー実行機能未作成のため、一覧表示不要
  - レビュースペースのサイドバーの各スペースは折りたたみボックスとなっており、展開するとチェックリストが確認できる
    - 展開せずにレビュースペース部分をクリックするとレビュー対象一覧画面に飛ぶ
  - チェックリスト表示画面（`10-checklist-view-edit`参考）にて手動でチェックリストを編集（項目追加、編集、削除）でき、「変更を保存」ボタンで一括保存できる
  - チェックリスト表示画面で複数チェック項目を選択した状態で、全て選択の横の削除ボタンを押すと、それらの項目が全て削除される
  - あるチェックリストを選択してから、`shift`キーを推しながら別のチェックリストを選択すると間のチェックリストが全て選択される

# ID: 2
- PBI名: チェックリストファイルインポート機能追加
- ステータス: to do
- ユーザストーリー
  - ユーザはレビュールームにチェックリストファイルをインポートすることができる
  - 既に運用されているチェックリストファイルをAIKATAでも利用することができるからだ
- 受け入れ基準
  - チェックリスト表示画面からチェックリストファイル（txt,csv,xlsx）のインポートができる
    - インポートの結果が即時反映される
- 注意事項
  - テキスト抽出処理を作成する必要があるはずなの、設計方針に沿って作成すること
  - インポート時はセルに改行を含む場合など様々な注意点があるためelectron版を十分に確認すること
- 指摘事項
  - `TextExtractor`がチェックリストファイルインポート機能を前提に作られている気がします。全体の設計を見直してください。
    - `TextExtractor`については今後のレビュー実行時のドキュメント処理にも利用したいので、汎用的な仕組みにして欲しいのです
    - 例えば`TextExtractorOptions`などは良い例です
      - これはテキスト抽出処理のオプションではなく、ファイルインポート機能のオプションです
    

# ID: 3

  - 出力したチェックリストを別のレビュールームで再インポートできる

# ID: 4
- PBI名: AIチェックリスト生成機能追加
- ステータス: to do
- ユーザストーリー
  - ユーザは、ドキュメントとチェックリスト生成要件をしていすると、AIがドキュメントからチェックリストを自動生成することができる
  - 例えば、ガイダンスを提供するようなドキュメントからチェック観点を抽出するなどするためだ
- 受け入れ基準
  - AIチェックリスト生成の画面が作成されている（`09-checklist-ai-generate`参考）
  - 複数ファイルをまとめてアップロードすることができる
    - pdfファイルの場合は画像として読み込ませることができる
      - pdfファイルはelectron版と同じようにブラウザで変換される
      - 複数pdfファイルをまとめて画像変換することができる
        - 大量のファイルがアップロードされた場合に一括指定できるため便利
  - チェックリスト生成結果はelectron版と同様の処理結果になる
    - 入力されたドキュメントとチェックリスト生成要件に対して、チェックリスト生成wokflowの出力が同一になる
- 注意事項
  - workflowはelectorn版を十分に参考にすること
    - ただし、既存の処理をそのままコピーすれば良いということではなく、workflow作成方針に沿って、必要あればリファクタリングすること
    - web版とelectron版では機能面や設計思想の違いがあるため、workflowの細かい設計について変更が必要な場合もあるだろう
      - その場合は、無理にelectorn版に寄せずに、web版の設計思想に基づいて実装すること
  - ファイルアップロードについてはレビュー実行でも利用するため共通コンポーネントとすること
